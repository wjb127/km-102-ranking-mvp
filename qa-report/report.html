A. 홈 / 네비게이션 PASS

다크 테마 히어로, 실시간 캐러셀(공지·HOT 게시글·신규 선수·이벤트), 4 quick action 카드(선수/일정/투표/게시판), 인기 선수 TOP 3, 예정 경기, 실시간 게시판 모듈. 비로그인 상태에서 로그인/회원가입 노출.

B. GOAT 투표 (계약 ①-1) PASS 경미한 버그

5개 카테고리(P4P / 헤비급 / 라이트급 / KO 아티스트 / 서브미션 아티스트). DB 기반 동적 후보 5명 노출. 카테고리당 1표, 토글 취소 가능.

발견 이슈

  • 버그 04 화면에서 "전체 투표 0표 참여중" 표시 — 1표 들어갔는데 합계 미갱신.
  • UX 카테고리별 후보 한글/영문 번역 일관성 부족 (P4P/KO는 영문, 헤비급은 한글).

C. 선수 검색 / 프로필 / 댓글 (계약 ①-2) PASS 데이터 갭

전체 8089명 중 60명 표시 + "더 보기" 페이지네이션. 한글/영문 동시 검색, 체급/현역 필터, 정렬 옵션. 프로필 상세에 댓글 시스템.

발견 이슈

  • 중대 프로필의 "전적 (전체 합산)"이 0/0/0. 목록 카드는 22-6-0 표시. fighter_org_records 집계 로직과 fighters.career_* 컬럼 사이 연동 누락. /fights 엔드포인트 유료 티어라 실데이터 부재일 가능성도 있음.
  • 데이터 "최근 경기: 경기 기록이 없습니다" — 동일 원인.

D. 경기 일정 + 댓글 (계약 ①-7) PASS

예정 / 종료 분리, 단체 배지(UFC/PFL), 일시·장소·국가 표시. 상세 페이지에 일시·장소·단체·파이트 카드·댓글.

발견 이슈

  • 데이터 파이트 카드 비어있음 — balldontlie /fights 엔드포인트 유료. 계약 제2조 ②항의 "선수 프로필 / 전적 (초기): ufcstats.com 1회성 크롤링"으로는 fight 단위 데이터 미확보.

E·F. 게시판 / 추천 / 이미지 (계약 ①-8, ①-9) PASS 시드 데이터

분석/토론/자유 분류 + 베스트/실시간 탭. 글쓰기 모달(카테고리·닉네임·제목·내용·이미지 첨부 최대 3장). 추천·조회수·댓글 카운트.

발견 이슈

  • 데이터 목록의 [89] 댓글 카운트는 시드 더미값. 상세 페이지에서는 "아직 댓글이 없습니다" 노출 — 시드 댓글 미적재.

G·H. 회원가입 / 로그인 / 쪽지 (계약 ①-10) PASS UX

이메일·닉네임·비밀번호 가입 → 자동 로그인. 쪽지함(받은/보낸) + 모달 작성. 본인 보호/대상 검증.

발견 이슈

  • 중대 회원가입/로그인 페이지가 다크 모드 미지원 — 흰 배경 카드, 나머지 페이지는 다크. 테마 일관성 깨짐.
  • 제약 게시글 작성자는 비회원 닉네임이라 쪽지 발송 불가. 게시판 닉네임을 회원과 연결하지 않으면 1:1 쪽지 시나리오가 사실상 회원 끼리만 가능.

I. 관리자 보호 (계약 ①-5) PASS

비관리자 계정으로 /admin 접근 시 홈으로 리다이렉트.

미검증 항목

  • 관리자 계정 미보유로 인해 실제 보정 UI(전적/순위 수동 보정, 게시글 삭제) 동작은 미테스트.

J. 한국어 번역 (계약 ①-6) 부분

DeepL 배치(scripts/translate-batch.ts) 일부 적용. 헤비급/라이트급은 한글 풀세트, P4P 카테고리는 미번역, KO 아티스트는 혼재.

  • UI 라벨/네비/버튼은 100% 한글.
  • 선수명·이벤트명 번역률 60~70% 추정 — 배치 재실행 또는 우선순위 큐 필요.

역질문 (의문점)

  • 1. balldontlie /fights 유료 결제 여부 — 파이트 단위 결과/메서드/라운드는 유료 티어. 결제 예정인지, 아니면 ufcstats 추가 크롤링으로 대체할지 의사 결정 필요. 선수 프로필 0/0/0 문제 직결.
  • 2. 게시판 작성자와 회원 연결 — 현재 posts.author_nickname은 자유 입력. 1:1 쪽지를 게시글 작성자에게 보내려면 users.id FK 필요. 계약 ①-10 "회원 간 쪽지" 범위가 회원-회원만으로 한정되는지 확인.
  • 3. NextAuth vs 자체 JWT — 계약서 표현은 "NextAuth"인데 실 구현은 bcrypt + jose 기반 자체 JWT. 스키마(accounts/sessions 테이블) 흔적은 남아있음. 계약 표현 수정 vs NextAuth 마이그레이션 중 어느 쪽?
  • 4. 관리자 권한 부여 절차 — users.role = 'admin' 수동 SQL 외에 부여 흐름이 있는지. 클라이언트 데모/검수용 관리자 계정 발급 가능?
  • 5. 시드 데이터 정책 — 게시판 30개 글의 추천/조회/댓글 [N] 숫자가 의도된 시드인지, 댓글 본문은 의도적으로 비워둔 건지. 운영 전 정리 필요?
  • 6. 한글 번역 스코프 — 모든 8089명 + 모든 이벤트를 DeepL 배치 돌리면 비용/시간 어느 정도 예산? 부분 번역 유지 vs 전수 번역 어느 쪽 선호?
  • 7. 라이트/다크 모드 정책 — 시스템 prefers-color-scheme 자동 감지 + 토글 두 가지 다 지원하는데, 회원가입/로그인 페이지만 라이트 고정인 건 의도? 다크가 default라면 수정 권장.
  • 8. 상용 트래픽 기준 레이트 리밋 — 익명 업로드 10/10분, 로그인 30/10분이 in-memory Map. 다중 인스턴스 환경(Vercel)에서 세션 간 일관성 깨짐. KV/Redis 도입 시점?
