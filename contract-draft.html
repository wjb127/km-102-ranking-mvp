제1조계약의 목적

본 계약은 갑이 을에게 MMA(종합격투기) 분석 커뮤니티 웹사이트 개발을 위탁하고, 을이 이를 성실히 수행하여 갑에게 납품함에 있어 양 당사자의 권리와 의무를 정함을 목적으로 한다.

제2조용역의 범위

① 공통 포함 기능
  • 역대 1위 투표 (GOAT 투표) 기능 — 선수 프로필 카드와 양방향 연동 (카드 클릭 시 상세 페이지 이동, 투표 결과는 선수별로 누적 표시)
  • 선수 검색 · 선수 프로필 페이지 · 선수별 댓글 기능
  • 선수 전적 누적 관리 — 선수가 타 단체(UFC → ONE, Bellator 등)로 이적하는 경우 단체별 경기 기록을 분리하여 누적 관리
  • 경기 결과 자동 반영 — 외부 API가 제공하는 무효 · 판정 번복 정보에 따라 선수 전적 자동 복귀 처리
  • 관리자 페이지 수동 보정 기능 — 외부 API 오류 또는 누락에 대비하여 관리자가 경기 결과 · 전적을 직접 수정할 수 있는 2차 안전장치 제공
  • 한국어 번역 기능 — 선수 프로필 · 경기 정보 등 외부 API 원문(영문) 데이터를 번역 API를 통해 한국어로 노출
  • 경기 일정 페이지 · 경기별 댓글 기능
  • 게시판 기능 — 분석 · 토론 · 자유 카테고리 분류 및 전체 목록 통합 조회(필터링 포함)
  • 게시글 기본 기능 — 조회수, 추천, 사진 업로드 (Cloudinary 연동)
  • 회원가입 · 로그인 · 회원 간 1:1 쪽지 기능
  • 반응형 UI (모바일 · 태블릿 · 데스크탑 대응)
② 데이터 구축 · 운영 방식 (B안)
구분 방식
선수 프로필 / 전적 (초기) ufcstats.com 1회성 크롤링을 통한 초기 DB 일괄 구축
선수 프로필 / 전적 (이후 갱신) balldontlie MMA API 연동을 통한 이후 경기 결과 누적 저장
경기 일정 balldontlie MMA API를 통한 자동 연동
게시판 · 회원 · 쪽지 · 투표 Neon(PostgreSQL) 기반 자체 데이터베이스 구축
※ 경기 무효 · 취소 처리의 경우, 외부 API가 해당 정보를 제공하는 범위 내에서 전적이 자동 복귀된다.
③ 외부 서비스 연동
  • MMA 데이터 API — balldontlie (유료 플랜, 갑이 발급·구독)
  • 이미지 저장 · 변환 — Cloudinary (갑이 계정 발급, 무료 플랜 시작 가능)
  • 데이터베이스 — Neon (서버리스 PostgreSQL, 갑이 계정 발급)
  • 회원 인증 — Auth.js(NextAuth) 기반 자체 구현 (별도 인증 서비스 비용 없음)
  • 호스팅 · 배포 — Vercel (갑이 계정 발급)
④ 미포함 사항
  • 영상 업로드 · 인코딩 · 스트리밍 기능 (유튜브/인스타 링크 임베드는 포함)
  • 모바일 앱(iOS/Android) 네이티브 빌드
  • 결제 · 수익화 · 배너 광고 기능
  • 실시간 채팅 · 푸시 알림 (쪽지는 포함, 별도 푸시 알림은 제외)
  • 본 계약서 제2조 ①~③항에 명시되지 않은 그 외의 기능

제3조개발 기간

개발 기간은 개발 시작일로부터 3주 ~ 4주로 한다.
※ 갑의 피드백 지연, 외부 API 장애, 요구사항 변경 등 을의 귀책이 아닌 사유로 발생한 지연은 위 기간에 포함되지 아니한다.

제5조산출물 및 소유권 이전

① 을은 최종 납품 시 다음 산출물을 갑에게 제공한다.
  • 완성된 웹사이트 (배포 URL 포함)
  • 전체 소스코드 (Git 저장소 접근 권한 또는 압축 파일 형태)
  • Neon · Vercel · Cloudinary 등 외부 서비스 계정 설정 인수 자료
  • 관리자 페이지 사용 가이드
② 본 계약에 따라 제작된 산출물의 저작권 및 소유권은 잔금 지급 완료 시점에 갑에게 완전히 이전된다.
③ 단, 을이 사전에 보유하고 있던 범용 코드 · 라이브러리 · 템플릿과 오픈소스 라이선스가 적용된 구성요소는 각 라이선스 조항에 따르며, 본조 ②항의 대상에서 제외된다.

※ 상기 비용은 각 서비스 제공자의 요금 정책에 따라 변동될 수 있으며, 을은 이에 대한 책임을 지지 아니한다.

제7조추가 요청 및 변경

① 본 계약서 제2조에 명시되지 아니한 기능의 추가 · 대폭적인 변경 요청은 별도 견적 및 합의에 따라 차액 결제 방식으로 진행한다.
② 경미한 수정(문구 변경, 색상 조정 등)은 갑의 검수 기간 내 무상으로 반영한다.
③ 기능 변경 요청 발생 시 을은 예상 소요 시간 · 추가 비용 · 일정 변동을 서면(카카오톡, 이메일 등 포함)으로 갑에게 고지하고, 갑의 서면 승인 후 작업을 진행한다.

제8조갑과 을의 의무

① 갑의 의무
  • 계약 금액을 제4조의 일정에 따라 지급한다.
  • 외부 서비스(Neon · balldontlie · Cloudinary · Vercel 등) 계정 발급 및 결제 수단 등록을 수행한다.
  • 개발에 필요한 자료(로고, 문구, 이미지, 엑셀 데이터 등)를 지체 없이 제공한다.
  • 을의 요청 사항 및 중간 산출물에 대해 영업일 기준 2일 이내 회신한다.
② 을의 의무
  • 본 계약서 제2조의 범위에 따라 성실히 개발을 수행한다.
  • 주요 마일스톤마다 갑에게 진행 상황을 공유한다.
  • 납품 후 제9조에 따른 하자보수 의무를 이행한다.
  • 갑에게 전달된 모든 자료 · 계정 정보를 안전하게 관리한다.

제9조유지보수 및 하자보수

① 최종 납품일로부터 30일간 을은 본 계약서 제2조의 범위 내에서 발생한 버그 · 오류를 무상으로 수정한다.
② 상기 기간 경과 후 또는 기능 추가 · 외부 API 스펙 변경에 따른 재작업은 별도 유상 유지보수 계약으로 진행한다.
③ 다음의 경우는 하자보수 범위에서 제외된다.
  • 갑 또는 제3자가 소스코드를 임의로 수정하여 발생한 오류
  • 외부 서비스 제공자의 장애 · 정책 변경에 따른 문제
  • 갑이 제공한 자료의 오류로 발생한 문제

제10조비밀 유지

갑과 을은 본 계약의 이행 과정에서 취득한 상대방의 영업비밀 · 개인정보 · 기술자료 등을 계약 기간 및 계약 종료 후에도 상대방의 사전 서면 동의 없이 제3자에게 누설하거나 본 계약 이외의 목적으로 사용할 수 없다.

제11조계약의 해지

① 일방이 본 계약을 중대하게 위반하고 상대방의 서면 시정 요청 후 7일 이내에 시정하지 아니할 경우, 상대방은 본 계약을 해지할 수 있다.
② 갑의 귀책사유로 계약이 해지된 경우, 을은 이미 수행한 업무 분량에 해당하는 금액을 청구할 수 있다.
③ 을의 귀책사유로 계약이 해지된 경우, 을은 이미 수령한 계약금 중 미수행분에 해당하는 금액을 갑에게 반환한다.

제12조지연배상 및 면책

① 을의 귀책사유로 납품이 7일 이상 지연될 경우, 지연 일수 1일당 계약 금액의 0.1%를 갑에게 지연배상금으로 지급한다. 단, 총 지연배상금은 계약 금액의 10%를 초과하지 아니한다.
② 천재지변 · 전쟁 · 팬데믹 · 외부 서비스 장애 등 불가항력 사유로 인한 지연은 면책된다.

제13조분쟁 해결

① 본 계약과 관련하여 분쟁이 발생할 경우 양 당사자는 상호 신의성실의 원칙에 따라 협의하여 해결한다.
② 협의가 이루어지지 아니할 경우, 서울중앙지방법원을 제1심 관할 법원으로 한다.

제14조기타

① 본 계약서에 명시되지 아니한 사항은 관련 법령 및 상관례에 따른다.
② 본 계약의 변경은 양 당사자의 서면 합의(이메일, 카카오톡 등 전자적 기록 포함)에 의해서만 효력이 발생한다.
③ 본 계약서는 2부를 작성하여 갑과 을이 각 1부씩 보관한다. 전자 서명 또는 스캔본의 경우 원본과 동일한 효력을 가진다.
