4. 외부 서비스 인수 절차

Neon (PostgreSQL)

  • 콘솔: console.neon.tech
  • 인수: 클라이언트 이메일 멤버 추가 → Owner 권한 이전 → 기존 계정 제거
  • 백업: Neon Branching으로 매일 자동 스냅샷. 추가 백업 pg_dump $DATABASE_URL > backup.sql
  • 환경변수: DATABASE_URL (Pooled connection)

Vercel (배포)

  • 콘솔: vercel.com
  • 인수: 팀 전환 또는 프로젝트 Transfer. 도메인은 Project → Domains
  • 환경변수: Project → Settings → Environment Variables (Production / Preview / Development)

Cloudinary (이미지 업로드)

  • 콘솔: cloudinary.com/console
  • 인수: Settings → Account → Change owner email
  • 환경변수: CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET

balldontlie (MMA API)

  • 콘솔: balldontlie.io/account
  • 요금제: ALL-STAR 이상 (MMA 엔드포인트 접근). 결제 카드 클라이언트 명의 변경
  • 환경변수: BALLDONTLIE_API_KEY

Anthropic / DeepL (선택)

  • 일괄 번역 스크립트 사용. 운영 중 직접 호출은 없음 (필요 시 수동 실행)
  • 환경변수: ANTHROPIC_API_KEY, DEEPL_API_KEY

6. 자동 동기화 (Vercel Cron)

주기경로대상
매일 18:00 UTC/api/cron/sync-mma선수 · 이벤트 최신 5페이지
매주 일요일 19:00 UTC/api/cron/sync-fights경기 결과 5페이지 + 전적 재계산

수동 트리거:

curl -H "Authorization: Bearer $CRON_SECRET" \
     "https://km-102-ranking-mvp.vercel.app/api/cron/sync-mma"

전체 풀 sync는 운영 안정화 이후 수개월 1회 정도 권장. 로컬에서 pnpm exec tsx scripts/sync-mma-full.ts 실행.

7. 관리자 페이지 사용법

/admin 접근 (role=admin 사용자 한정). 사이드바 메뉴:

메뉴경로기능
대시보드/admin가입자 · 게시글 · 신고 통계
회원 관리/admin/users검색 · 밴 · role 부여
선수 수정/admin/fighters한국어 표기 · 체급 · 키 · 국적 · 닉네임
선수 전적/admin/fighters/[id]단체별 W/L/D/NC 직접 보정 (감사 로그)
경기 보정/admin/fights승자 · 결과 · 방법 · 라운드 · 시간 · 무효
게시글 관리/admin/posts신고 누적 게시글 숨김 / 복구
신고 처리/admin/reports댓글 신고 검토 → 처리 또는 무시
보정 이력/admin/overridesadmin_overrides 전체 감사 로그

관리자 권한 부여 (최초 1회)

UPDATE users SET role = 'admin' WHERE email = '<관리자_이메일>';

이후 /admin/users에서 다른 회원에게 부여 가능.

신고 자동 숨김 임계값

  • 댓글: 3회 신고 → is_deleted = true
  • 게시글: 5회 신고 → hidden_by_admin = true

관리자가 /admin/reports에서 사실 확인 후 영구 처리 또는 복구.
