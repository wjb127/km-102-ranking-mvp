2. 구현 기능 상세

자동 동기화 balldontlie API에서 선수·이벤트·경기 데이터를 페이지네이션으로 수집합니다. cursor 기반 13초 rate limit 준수. 충돌 시 upsert 처리.
한국어 음차 Anthropic Claude Haiku 4.5 일괄 음차 적용. 선수 3,842명, 닉네임 2,820건, 경기장 307건 처리 완료.
통합 검색 /api/search 엔드포인트로 선수·이벤트·체급 횡단 검색. 한글 입력으로도 영문 원문이 매칭됩니다.
GOAT 투표 /vote 페이지에서 체급별·파운드포파운드 투표. 비로그인은 fingerprint, 로그인은 userId 기반 중복 방지.
관리자 보정 선수 정보 / 단체별 W·L·D 합계 / 경기 결과를 직접 보정합니다. 모든 변경은 admin_overrides 테이블에 before·after 전체 기록.
게시판 · 댓글 카테고리: 분석 / 토론 / 자유. Cloudinary 서명 업로드, 추천, 조회수, 폴리모픽 댓글 (게시글·선수·이벤트·경기 부착).
신고 · 숨김 댓글 3회 누적 시 자동 숨김. 게시글 5회 누적 시 hidden_by_admin 자동 처리. 관리자가 사후 검토 후 영구 처리 또는 복구.
인증 · 권한 자체 JWT 세션 (HttpOnly 쿠키). role: user / admin. 관리자 API는 requireAdmin 가드로 보호.

4. 외부 서비스 인수 절차

Neon (PostgreSQL)

  • 콘솔: console.neon.tech
  • 인수: 클라이언트 이메일을 프로젝트 멤버로 추가 → Owner 권한 이전 → 기존 계정 제거
  • 백업: Neon Branching으로 매일 자동 스냅샷 생성. 추가 백업은 pg_dump $DATABASE_URL > backup.sql
  • 환경변수: DATABASE_URL (Pooled connection 문자열)

Vercel (배포)

  • 콘솔: vercel.com
  • 인수: 팀 전환 또는 프로젝트 Transfer 사용. 도메인은 Project → Domains
  • 환경변수: Project → Settings → Environment Variables (Production · Preview · Development 각각)

Cloudinary (이미지 업로드)

  • 콘솔: cloudinary.com/console
  • 인수: Settings → Account → Change owner email
  • 환경변수: CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET

balldontlie (MMA 데이터 API)

  • 콘솔: balldontlie.io/account
  • 요금제: ALL-STAR 이상 (MMA 엔드포인트 접근 권한). 결제 카드를 클라이언트 명의로 변경
  • 환경변수: BALLDONTLIE_API_KEY

Anthropic / DeepL (선택 사항)

  • 일괄 번역 스크립트 전용. 운영 중 직접 호출은 없음 (필요 시 수동 실행)
  • 환경변수: ANTHROPIC_API_KEY, DEEPL_API_KEY

6. 자동 동기화 (Vercel Cron)

주기경로대상
매일 18:00 UTC/api/cron/sync-mma선수 · 이벤트 최신 5페이지
매주 일요일 19:00 UTC/api/cron/sync-fights경기 결과 5페이지 + 전적 재계산

수동 트리거

curl -H "Authorization: Bearer $CRON_SECRET" \
     "https://km-102-ranking-mvp.vercel.app/api/cron/sync-mma"

전체 풀 sync는 운영 안정화 이후 수개월 1회 정도 권장합니다. 로컬에서 pnpm exec tsx scripts/sync-mma-full.ts 명령으로 실행 가능합니다.

7. 관리자 페이지 사용법

/admin 경로 접근 (role=admin 사용자 한정). 사이드바 메뉴는 다음과 같습니다.

메뉴경로기능
대시보드/admin가입자 · 게시글 · 신고 통계
회원 관리/admin/users회원 검색 · 밴 · role 변경
선수 수정/admin/fighters한국어 표기 · 체급 · 키 · 국적 · 닉네임
선수 전적/admin/fighters/[id]단체별 W/L/D/NC 합계 직접 보정 (감사 로그)
경기 보정/admin/fights승자 · 결과 · 방법 · 라운드 · 시간 · 무효 처리
게시글 관리/admin/posts신고 누적 게시글 숨김 / 복구
신고 처리/admin/reports댓글 신고 검토 → 처리 또는 무시
보정 이력/admin/overridesadmin_overrides 전체 감사 로그

관리자 권한 부여 (최초 1회)

UPDATE users SET role = 'admin' WHERE email = '<관리자_이메일>';

이후에는 /admin/users 메뉴에서 다른 회원에게 직접 부여할 수 있습니다.

신고 자동 숨김 임계값

  • 댓글: 3회 신고 → is_deleted = true
  • 게시글: 5회 신고 → hidden_by_admin = true

관리자가 /admin/reports에서 사실 확인 후 영구 처리하거나 복구합니다.

8. DB 마이그레이션

신규 컬럼 / 테이블 추가 시 db/migrations/ 디렉토리에 SQL 파일을 생성한 뒤 적용합니다.

psql "$DATABASE_URL" -f db/migrations/000X_xxx.sql

현재 적용된 마이그레이션

  • 0001_fighter_career_fields.sql — 선수 커리어 컬럼 추가
  • 0002_add_missing_api_fields.sql — API 누락 필드 추가
  • 0003_admin_profile.sql — 관리자 / 프로필 관련
  • 0004_board_post_reports.sql — 게시글 신고 테이블
